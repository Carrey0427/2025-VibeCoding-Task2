關於我（About 區占位描述）

這裡目前是 About 區的占位內容，預計用來介紹開發者的背景、工作年資、專長領域與簡短的職涯重點。 之後可以補上更完整的自我介紹、價值觀與合作風格，讓訪客快速理解你是怎樣的一位前端工程師。

未來會使用 Section SDD 的 Prompt 產生新的 About section， 可能會包含多段落敘述、時間軸或列表介紹等版面。屆時請把新的 section 程式碼整段貼在「SECTION:ABOUT-START」與「SECTION:ABOUT-END」這兩個註解中間， 完全覆蓋現在這一區的占位內容。

開發者提示：若之後要加入更多區塊（例如時間軸、學經歷列表），建議維持 class-based 的寫法，避免改動既有 id， 以確保導覽列的錨點連結（href="#about"）仍然可以正常捲動到這個區域。

技能（Skills 區占位描述）

這裡目前是 Skills 區的占位內容，預計用來整理前端技術與相關能力， 例如：HTML、CSS、JavaScript、前端框架、測試、版本控制與團隊協作工具等。

接下來若使用 Section SDD 的 Prompt 產生新的 Skills section， 可以設計為技能標籤、分組列表或等級說明。完成後請將新的 section 程式碼整段貼在「SECTION:SKILLS-START」與「SECTION:SKILLS-END」註解中間， 完整取代目前這一區的占位文字與簡易排版。

前端基礎（HTML / CSS / RWD）占位

前端框架與工具（示例：React、Vue）占位

開發者提示：正式的 Skills 版面可以包含技能分級、常用技術堆疊（tech stack）與工具列表。 產出後請直接以新的 section id="skills" class="section section-skills">… 整段覆蓋這兩個註解之間的內容。

作品集（Projects 區占位描述）

這裡目前是 Projects 區的占位內容，預計用來展示代表性的前端作品， 例如個人 side project、實際專案經驗、開源貢獻或 UI 作品展示等。

未來透過 Section SDD 的 Prompt 產生新的 Projects section 時， 可以將每個作品設計為卡片、列表或畫廊形式。完成後請把新的 section 程式碼整段貼在「SECTION:PROJECTS-START」與「SECTION:PROJECTS-END」註解之間， 完全取代這裡的占位文案與示意卡片。

作品卡片占位 A

之後可以放上作品名稱、技術堆疊（例如：HTML / CSS / JS）與簡短說明， 並附上連結前往實際 Demo 或 GitHub 儲存庫。

占位說明：這裡只是用來提醒開發者未來可以放入 1～2 行的作品亮點，例如「負責前端架構設計與 RWD Layout」、 「實作深色模式切換」等內容。

作品卡片占位 B

這張卡片同樣為占位用途，可以預留給另一個類型不同的專案，例如後台管理系統、資料視覺化儀表板或 Landing Page。

開發者之後可在 Section SDD 生成的 Projects 內容中，改成實際的專案標題、專案描述與外部連結。 本區的所有卡片都可以被完整替換掉。

開發者提示：若未來要增加更多作品項目，建議仍沿用 .project-card 這類 class， 讓新舊設計之間的 CSS 可以平順過渡。真正替換時，請記得只在「SECTION:PROJECTS-START」與「SECTION:PROJECTS-END」註解之間貼入新的 section。

聯絡我（Contact 區占位描述）

這裡目前是 Contact 區的占位內容，預計用來放上 Email、社群帳號、GitHub 或 LinkedIn 等聯絡方式， 方便雇主、同事或合作夥伴與你取得聯繫。

未來可以透過 Section SDD 的 Prompt 產生更完整的 Contact section， 例如加入簡短的合作說明、回覆時間或表單連結。完成後請將新的 section 程式碼整段貼在「SECTION:CONTACT-START」與「SECTION:CONTACT-END」註解中間， 完全取代現在這一區的占位文字與示意連結。

開發者提示：請維持這個 section 的 id="contact" 不變，以確保導覽列的錨點 href="#contact" 仍能正確捲動到聯絡區。
